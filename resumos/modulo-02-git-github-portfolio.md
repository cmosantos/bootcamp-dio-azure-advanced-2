# Módulo 2 – Git, GitHub, Portfólio e Open Source

## 1. Fundamentos de Git

- Conceitos principais:
  - repositório local e remoto
  - commit, branch, merge
- Fluxo básico utilizado:
  1. `git clone` – clonar repositório
  2. criar branch para alteração
  3. editar arquivos
  4. `git add` → `git commit` → `git push`

## 2. GitHub na prática

- Repositórios públicos x privados
- Pull Requests:
  - revisão de código
  - comentários
  - aprovação/merge
- Issues:
  - organização de tarefas
  - registro de bugs e melhorias

## 3. Portfólio vencedor

- Portfólio não é só lista de cursos, e sim **projetos reais**
- Cada repositório deve mostrar:
  - contexto
  - problema
  - solução proposta
  - tecnologias utilizadas
- Importância de:
  - README bem feito
  - commits claros
  - organização geral da conta GitHub

## 4. Contribuindo em projetos open source

- Fluxo praticado no desafio:
  1. fazer **fork** do repositório
  2. clonar o fork
  3. criar uma branch específica
  4. implementar alteração (código ou docs)
  5. `git commit` + `git push`
  6. abrir **Pull Request** para o repo original
- Boas práticas vistas:
  - ler README e CONTRIBUTING antes de alterar
  - seguir padrão de código do projeto
  - ser objetivo na descrição da PR

## 5. Conclusão

Esse módulo consolidou:
- uso diário de Git e GitHub
- visão de como construir um portfólio forte
- primeiros passos reais em contribuição open source
